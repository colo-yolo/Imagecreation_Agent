# 团队展示场景规律

本参考提炼自团队上传的图片创作记录、场景模板和去重后的视觉样本。只保留跨案例稳定的规律；不要使用记录中的 API 配置、个人信息或具体字迹请求。

## 核心构图契约

将展示背景拆成五个必须分别描述的区域：

1. **预留区**：明确是全背景、矩形门洞、圆形月洞、拱门内部还是中央上方区域。要求纯黑、无纹理、无雾、无渐变、无光点、无装饰穿越。
2. **地台**：低矮、居中、正对镜头、表面空置，通常位于画面下方约五分之一到四分之一。
3. **边缘布景**：植物、建筑、机械或自然地貌集中在左右、上缘和下缘，不侵入预留区。
4. **照明区**：纯黑预留区保持不受光；地台和边缘布景仍使用白天自然光、摄影棚面光或清晰工业光。
5. **安全边界**：不要裁切地台，不让前景遮挡地台，不让枝叶、机械臂、光束或粒子进入预留区。

默认采用 16:9、正面平视、中心轴或左右平衡构图。用户指定其他设备比例时覆盖默认值。

## 场景族

| 场景族 | 保持项 | 可变化项 | 主要风险 |
| --- | --- | --- | --- |
| 花园水面舞台 | 黑色预留区、低圆台、清澈水面、边缘花园 | 拱门形状、花种、季节、桥与温室结构 | 花叶过密、倒影污染黑区 |
| 机库与维护舱 | 黑色门洞、低维护台、左右机械结构、明亮工业光 | 装甲语言、挂载件、灯带、机械臂 | 整体过暗、机械侵入中心、零件堆砌 |
| 自然地台 | 黑色预留区、浅色地台、边缘自然景观 | 树种、岩石、水面、苔藓、季节 | 树干过粗、枝叶挡中心、脏色 |
| 东方圆门 | 圆门内部纯黑、下方圆台、水面、平视 | 门框材质、庭院元素、花木与季节 | 圆门变形、雾或花瓣进入门内 |
| 极端环境舞台 | 中央黑区、完整地台、边缘大型地貌 | 冰雪、沙漠、火山、云团、晶体 | 气势被误写成杂乱，烟雾遮挡中心 |
| 黑底单体素材 | 单一主体完整、纯黑全背景、轮廓清晰 | 材质、配色、能量效果 | 被误生成完整场景或裁切主体 |

## 将模糊反馈转为可执行修改

| 用户反馈 | 改写为 Prompt 约束 |
| --- | --- |
| 太黑、太脏 | 只让预留区纯黑；提高前景与边缘曝光；使用白天或摄影棚光；减少灰雾和低饱和脏色 |
| 地平线或地台太高 | 将地台表面降到画面下方约五分之一；使用正面平视或轻微低机位；增加上方负空间 |
| 中间不够干净 | 重复预留区形状和禁入内容；把装饰、光束、雾气和粒子移动到边缘 |
| 细节太多 | 限制道具种类与数量；保留前景、地台、边缘布景三层；删除微小散件和重复纹理 |
| 摄像机太高或太斜 | 指定正面平视、镜头光轴水平、无俯视、无斜视、地台正对镜头 |
| 太卡通、没层次 | 改为动画电影背景或高级卡通 3D；增加材质层次、柔和阴影和景深，不必直接改成写实摄影 |
| 不够宏伟 | 增加高耸边框、宽阔开间、远近尺度和小型尺度参照；保持中心留空，不靠堆零件制造规模感 |
| 去掉主体但保留地台 | 明确删除地台上方的主体及相关影子，只保留空地台、构图、背景和灯光 |
| 其他不变 | 在新 Prompt 中逐项重复构图、视角、黑区、地台、风格、照明和未修改区域 |

## 批量变化矩阵

先锁定黑区形状、地台位置、镜头、画幅、照明分离和风格。再从下列变化轴中每张选择两到三个：

- 空间轮廓：圆拱、方门、月洞、温室、石庭、维护舱。
- 边缘内容：花种、树种、机械臂、挂载件、岩石或晶体。
- 地面与地台：浅水、石板、金属、玉石、冰面或自然岩石。
- 氛围：晴日、秋日、月光、轻雾、冷工业光或暖边缘光。
- 色彩：保持一个主色、一个辅色和一个小面积点缀色。

不要用“每张随机不同”替代变化设计。先列出每张的变化组合，再分别编写 Prompt。

## 检查清单

- 黑区像素语义是否明确且没有光、雾、纹理或物体穿越？
- 地台是否低矮、完整、空置并且正对镜头？
- 视角是否平视，地平线是否足够低？
- 边缘场景是否清晰明亮但不过密？
- 风格、材质和色彩是否统一且没有脏色？
- 批量作品是否保留共同契约，同时在可控维度上真正不同？
