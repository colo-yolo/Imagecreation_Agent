# 视频 Prompt 工作流

截至 2026-07-20，上传的团队案例没有完整的视频生成记录。本参考提供供应商中立的保守工作流；不要把这里的建议描述成团队实测结论。

## 选择生成方式

- 用户已有满意图片并希望“让它动起来”时，采用图生视频，图片标记为 `首帧` 和 `构图参考`。
- 用户只给场景想法时，采用文生视频。
- 用户要求首尾一致、无缝播放或展示柜常驻内容时，采用循环视频契约。

只在缺失信息会改变成片时询问：时长、画幅、镜头是否移动、是否循环。平台未指定时不要编造模型参数。

## 编译结构

```text
用途：<展示柜循环 / 宣传片段 / 场景氛围 / 其他>
生成方式：<文生视频 / 图生视频>
起始画面：<第一帧的主体、构图和环境>
主体动作：<一个清晰、可完成的主要动作>
环境运动：<最多两个次要运动>
镜头：<固定、缓慢推近、横移、环绕等>
时间推进：<开始 -> 中段 -> 结束>
结束状态：<最终姿态或与首帧衔接方式>
灯光与色彩连续性：<不得漂移的曝光和色调>
参数：<时长、画幅、帧率或平台支持的参数>
必须保持：<身份、构图、预留区、材质和场景结构>
避免：<闪烁、形变、穿模、突然切镜、额外物体等>
```

## 动作预算

默认只安排一个主要动作和最多两个环境动作。例如：主体缓慢转头；粒子轻微上升；水面产生细小波纹。动作越多，主体和构图越容易漂移。

不要把“动态、震撼、电影感”当成动作说明。写出动作方向、速度、幅度、开始时机和结束状态。

## 图生视频不变量

重复说明：保持首帧的主体身份、轮廓、比例、服装或装甲、相机位置、构图、地台、背景结构和色彩。只让用户指定的元素发生运动。

用于实体模型背景时，保护纯黑预留区：不让粒子、雾、光束、枝叶、机械臂、倒影或镜头运动进入该区域。镜头移动时也要保持预留区形状和位置稳定。

## 循环展示默认值

用户未指定且用途是展示柜循环时，建议 5 至 8 秒、固定镜头或极慢推近，并使结束状态自然回到起始状态：

- 粒子沿闭合路径缓慢流动。
- 水波幅度在结尾回到接近首帧。
- 灯带亮度缓慢呼吸，但首尾亮度一致。
- 不使用切镜、快速变焦、突然爆发或穿越黑区的物体。

## 常见失败修正

| 问题 | 修正 |
| --- | --- |
| 主体变形或身份漂移 | 减少动作幅度，锁定轮廓与关键特征，缩短镜头运动 |
| 画面闪烁 | 锁定曝光、白平衡、光源位置和材质，删除频繁脉冲光 |
| 黑区被污染 | 锁定预留区蒙版语义，删除穿越该区的粒子、雾、光和反射 |
| 动作太乱 | 保留一个主要动作，删除不相关环境运动 |
| 循环接缝明显 | 让动作走闭合路径，并明确首尾姿态、镜头和亮度一致 |
| 镜头造成构图漂移 | 改为固定镜头，或限定极慢、短距离、单方向运动 |

## 交付

交付供应商中立 Prompt 和建议参数。只有用户指定平台且确认该平台支持相应字段时，才转换为平台专用语法。没有视频工具时明确说明仅完成 Prompt。
